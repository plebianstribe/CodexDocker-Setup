# Refinement Rubric

Use this rubric to evaluate whether the refinement is genuinely domain-specific and execution-ready.

## Scoring

Score each category from 1 to 5:
- 1: vague / generic
- 3: usable but incomplete
- 5: specific, testable, and decision-ready

## Categories

1. Domain fidelity
- Uses domain-native language correctly.
- Reflects real operational/regulatory constraints.

2. Persona realism
- Captures incentives, risk tolerance, and buying power of the persona.
- Addresses likely objections from that persona.

3. Value mechanism clarity
- Clearly explains how value is created and measured.
- Defines who pays, who benefits, and why now.

4. Implementation specificity
- Identifies components, integrations, and dependencies.
- Includes phased rollout with concrete milestones.

5. Risk and assumptions quality
- Lists critical assumptions explicitly.
- Provides mitigation and de-risking actions.

6. Validation design
- Proposes high-learning experiments.
- Includes pass/fail thresholds and decision rules.

## Minimum quality threshold

A strong output should score:
- no category below 3
- average >= 4

If below threshold, revise sections with the weakest scores before handoff.
